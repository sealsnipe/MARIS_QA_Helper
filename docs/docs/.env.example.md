# `.env.example`

**Quellpfad:** `.env.example`

## Zweck

Standard-Konfigurationsvorlage für Docker Compose und Produktions-Setup. Wird von `scripts/setup.py`, `scripts/setup_env.py` und der README als Ausgangspunkt für `.env` verwendet. Enthält Platzhalter für Secrets — niemals echte Keys committen.

## Ablauf (kurz)

1. `cp .env.example .env` (manuell oder via `./setup.sh` / `./install.sh`).
2. Pflichtwerte setzen: `SESSION_SECRET`, `OPENAI_API_KEY`.
3. Bei OAuth-Chat: `LLM_AUTH_MODE=chatgpt_oauth` und Codex-Auth-Dateien bereitstellen.
4. Docker Compose liest `.env` für Service `api` (siehe `docker-compose.yml`).

## Konfiguration / Parameter

| Variable | Beispielwert | Beschreibung |
|---|---|---|
| `APP_HOST` | `0.0.0.0` | Bind-Adresse des FastAPI-Servers |
| `APP_PORT` | `8088` | HTTP-Port (Compose mappt typisch 8088:8088) |
| `SESSION_SECRET` | `change-me-generate-with-secrets-token-urlsafe` | Pflicht: Session-Signatur |
| `DATABASE_URL` | `sqlite:///./data/support_kb.sqlite3` | SQLite-Pfad im Container/Host |
| `QDRANT_URL` | `http://qdrant:6333` | Qdrant-Service-URL in Docker |
| `COLLECTION_PREFIX` | `kb_` | Präfix für Mandanten-Collections |
| `OPENAI_API_KEY` | `sk-placeholder-replace-me` | Pflicht: API-Key für Embeddings/LLM |
| `OPENAI_BASE_URL` | `https://api.openai.com/v1` | OpenAI-kompatible API-Basis |
| `EMBEDDING_MODEL` | `text-embedding-3-small` | Embedding-Modell |
| `EMBEDDING_DIM` | `1536` | Embedding-Dimension |
| `LLM_AUTH_MODE` | `chatgpt_oauth` | `api_key` (Platform API) oder `chatgpt_oauth` (Codex/ChatGPT-Abo) |
| `CHAT_MODEL` | `gpt-5.4-mini` | Chat-/Agent-Modell |
| `CODEX_AUTH_PATH` | `~/.codex/auth.json` | OAuth-Auth bei `chatgpt_oauth` |
| `CODEX_OAUTH_AUTH_PATH` | `~/.oauth_codex/auth.json` | Alternative OAuth-Datei |
| `CODEX_BASE_URL` | `https://chatgpt.com/backend-api/codex` | Codex-Backend |
| `SESSION_COOKIE_SECURE` | `false` | Auf `true` setzen hinter HTTPS (`docker-compose.prod.yml`) |
| `DEPLOY_PROFILE` | `dev` | Wird von `./setup.sh` / `./install.sh` geschrieben |
| `TOP_K_DEFAULT` | `4` | Standard-Retrieval-Top-K |
| `MIN_SCORE_DEFAULT` | `0.25` | Mindest-Relevanzscore |
| `MAX_TOOL_ROUNDS` | `4` | Agent-Tool-Iterationen |
| `MAX_UPLOAD_MB` | `30` | Upload-Limit |
| `ALLOWED_EXTENSIONS` | `.txt,.md,.pdf,.docx` | Erlaubte Upload-Endungen |

Werte werden von `app.config.Settings` geladen (`.env` im Arbeitsverzeichnis).
